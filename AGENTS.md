# Codex Workspace Instructions

This workspace is a multi-project workbench. Do not assume the root directory is
the active application.

## Mandatory Read-First Files

Before planning, editing, deploying, importing, or recommending work in this
workspace, read:

1. `MEMORY.md`
2. `DECISIONS.md`
3. The target project's own `README.md`, handoff, or review document

## Required Work Pattern

1. Identify the active project folder.
2. Scope edits to that project unless the user explicitly asks for cross-project work.
3. Use concrete evidence from files, diffs, tests, screenshots, connector results, or source artifacts.
4. Keep draft gates before live writes, database imports, and public deploys.
5. Do not open `.env.local` or expose secrets.
6. Keep simple static/content projects simple unless the user asks for a fuller app.
